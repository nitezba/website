A group is a set $G$ that abides by certain properties. This is what I see in my head when I need to think about a group:

Call it my version of a mnemonic. Early on, the first step to understanding a group is to actually remember its properties (I confess to having given this too much thought, the picture keeps it simple for me). In order for a set to be a group, we first need a binary operator (also known as a law of composition) to act on its elements. Formally, this is a map $+:G \times G \to G$; and example is $+(a,b) = c$. This is just a rule that answers "what happens when we take element $a$ and mash is with element $b$?", or simply, $a+b = c$. So, if this map/binary operator/law of composition/operation has four golden properties, we call it a group:

  1. Closure - the operation will always yield another element of the set
  2. Associativity - parenthesize at will!
  3. Identity - there is some element of $G$ such that operating on it with any other element will yield that other element
  4. Inverses - for every element of $G$, there is another (not necessarily distinct) element such that operating on the two yields that special identity element from before
Boom. You know what group is. You probably feel like it's a very arbitrary and maybe even random set of things to define, but the motivation and importance comes with a bit of time.

A motivating example of a group is the dihedral group. The general definition is that it is the group of symmetries of a regular n-gon. Too un-jargon this, let's think about a pentagon. First though, let's think a little more formally about symmetry. Normally we probably think about symmetry in the context of "this shape is symmetric because if I move it around in $x$ or $y$ way, I can still get a shape that looks the same." To add some rigor, note that in order for a translation of the shape to be valid, adjacent vertices must stay adjacent. This is inherently tied to the idea of a line of symmetry.

Here if we rotate the pentagon such that all vertices fall on the location of previous vertices, we essentially have the same shape (this is a rotation by $\frac{360}{5} = 72$ degrees). If we do this three more times, we get all the distinct variations of the pentagon that preserve adjacent pairs before landing on where we started (remember this). This isn't enough though! If the image here can be written as $ABCDE$, then what about $AEDCB$? It preserves pairs like before, but isn't obtained through any rotation; hence we consider reflections as well. Now we can start to see a bit of the group structure that emerges naturally. Remember how rotating a certain number of times (5 in this case) puts the shape back at where it started - the same is true of reflecting it across the same axis twice. Look back at the group axioms from earlier and see if you can identify how this might be of importance to us.

What you probably should've thought first was "wait don't we need some kind of operation first before we can think about a group" and you're right! But we already kinda have it; the only thing missing is to observe that composing rotations and reflections is this operation. Note that it's not the act of applying the symmetries onto the polygon, but rather doing one after the other that gives us an operation - which you should decide for yourself is closed. To make that last bit easier, we can start to abstract away a bit: call the rotation of the pentagon $\rho$ and the reflection $\tau$. These can now be treated as elements of a set rather than translations of a shape. And they're special ones at that since they're going to generate the rest of our set.   • Relations are sets. Given a set $X$ and a relation $R$ on $X$, for a given $x \in X$, $R \subseteq X \times X$ is the set of all elements of $X \times X$ such that $xRy$ (i.e. $x$ is in the $R$ relation to $y$).
  • Given a relation, if it has three special properties (reflexivity, symmetry, and the transitive property) we call it an equivalence relation. This is usually given by ~ instead of $R$ though it can be read the same.
  • Similarly, if it has anti-reflexivity, the transitive property, and either $xRy$ or $yRx$ we call it an order relation (typically given by $<$).
  • Two important indexing sets are $\{1, \dots, m\}$ and $\mathbb{Z}^+$. The idea of indexing sets (along with m-tuples) helps define a more general notion of the cartesian product, though I'm not entirely sure how this works just yet.
